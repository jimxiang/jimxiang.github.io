2016-03-29 | AngularJS

AngularJS学习笔记

（一）.
题外话：最近由于项目需要，要做一个Angular应用，发现自己只会一些皮毛，深层次的东西完全摸不着头脑，深有拖team后腿的感觉，所以我准备利用闲暇时间好好学习AngularJS，也包括NodeJS，来增强自己的编程能力、扩展自己的编程思路。我会把每天学到的知识用自己的理解写进博客里，一是为了检验自己是否理解了，二是便于自己以后复习巩固。这篇博客不准备分篇了，就写在这一篇里面，我很期待到最后这篇博客能有多长。—2016.3.29

（二）.
今天遇到一个超级蛋疼又超级无语的bug，吃完晚饭后大概7点多一直折腾到11点！具体情况是我们的项目用AngularJS框架搭建，项目和广告有关，今天重构了一发后，队友增加了几个路由，其中几个以advert关键字命名，其他人的项目跑的很顺畅，唯有我的一直有bug，浏览器老是请求不到一些css和js资源，明明是从github上pull下来的同一份代码，为什么我的就有bug呢？我苦苦思索，用尽各种方法调试都不成功。最后迫不得已，百度了一发，因为我觉得这种资源无发请求的问题用不着上网查资料，不查不知道，一查吓一跳，在stackoverflow上有个人竟然和我有一样的问题，我仔细一看竟然连请求不道德资源名称都差不多，我们都是带有advert关键字的资源访问不到，一看下面的回答，我的妈呀，结果是应为我们都安装了屏蔽广告的chrome插件，这插件居然会阻止请求带有advert关键字的请求，因为它把这种请求当作广告了，真是日狗！！！还好找到了问题所在，不然以后都没法和队友协同工作了。这真是个不错的教训：1.不要乱装插件；2.做web开发，电脑里做好多装几个浏览器，也做好把工作的测试浏览器与平时使用的浏览器分开，或者使用多浏览器模拟器来进行开发。总之都是减少干扰因素。—2016.3.30

（三）.
Angular Services 是通过依赖注入(Dependency Injection)连接在一起的可替代的对象。你可以使用服务来组织和共享应用中的某些代码。

  1. Services的特点：
    . Services只有当有程序组件依赖时才会被Angular实例化
    . Services是单例模式。依赖于Services的组件会得到一个有Services Factory生成的实例。
    . 可自定义Services

2.Services的使用：
把Services的名字添加到依赖Services的组件的Controller的依赖注入数组中： 


angular.
module('module', []).
  controller('Controller', ['$scope', 'ServiceName', function($scope, ServiceName) {  
}]);

3.. 创建Services

    1. 注册Services 
      
var myModule =- angular.module('myModule', []);
myModule.factory('serviceId', function() {
var newServiceInstance;
return newServiceInstance;
});
    1. 依赖
      Services也有自己的依赖。

      var batchModule = angular.module(‘batchModule’, []);
      batchModule.factory(‘batchLog’, [‘$interval’, ‘$log’, function($interval, $log) {
      var messageQueue = [];

    function log() {
    if (messageQueue.length) {
    $log.log(‘batchLog messages: ‘, messageQueue);
    messageQueue = [];
    }
    }
    $interval(log, 50000);

    return function(message) {
    messageQueue.push(message);
    }
    }]);

batchModule.factory(‘routeTemplateMonitor’, [‘$route’, ‘batchLog’, ‘$rootScope’,
function($route, batchLog, $rootScope) {
return {
startMonitoring: function() {
$rootScope.$on(‘$routeChangeSuccess’, function() {
batchLog($route.current ? $route.current.template : null);
});
}
};
}]);

  • 3.使用$provide注册Services

    angular.module(‘myModule’, []).config([‘$provide’, function($provide) {
    $provide.factory(‘serviceId’, function() {
     var shinyNewServiceInstance;
     // factory function body that constructs shinyNewServiceInstance
     return shinyNewServiceInstance;
    });
    }]);
